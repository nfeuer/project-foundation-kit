# Changelog

All notable changes to the kit, newest first. Sections are generated by the
`release` skill from conventional commits (`docs/COMMIT_CONVENTION.md`).

## [v1.3.0] — 2026-07-06

Version jumps from v1.1.0 to v1.3.0 to track `kit_version`, which both features
below bumped once each.

### Features
- **observability:** logging standard covers every process type, not just API
  calls — per-process-type event catalog in `docs/LOGGING_STANDARD.md`,
  `log_operation` context manager, observability-check and reviewer-agent
  coverage for jobs/consumers/pipelines (#3)
- **skills:** `logging-init` — interactive, evidence-based logging
  initialization: probes existing logging, classifies the repo, asks batched
  questions, wires a stack-mapped setup additively; recorded in the new
  `logging` block of kit.yaml (#4)

## [v1.1.0] — 2026-07-04

First tagged release. Covers the full history to date; commits before the
conventional-commit convention was adopted are summarized under Other.

### Features
- **skills:** add 6 ecosystem-inspired improvements — `using-the-kit` (1%-rule
  dispatcher + trigger index), `compound-learnings` (docs/solutions/ loop),
  `writing-kit-skills` (authoring meta-skill), `config-audit` (config-surface
  security audit), `gates.strictness` (prototype/standard/production), and
  progress ledgers for long-running skills (#1)

### Fixes
- **ci:** stop template workflow running in kit repo; add real kit-ci that
  validates the kit's own artifacts

### Other
